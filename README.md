# operational-efficiency-ai-automation

**Type:** Workflow Automation | PM Operations  
**Tools:** Zapier, Claude (Anthropic), Jira, Slack, HubSpot  
**Role:** Associate Product Manager at Sprinto  
**Status:** Live and Running

---

## Overview

As a PM at Sprinto, I built a suite of automations using Zapier and Claude to eliminate repetitive operational tasks, improve cross-team visibility, and keep product workflows moving without manual intervention.

These automations connect Jira, Slack, and HubSpot to handle pre-development release workflows, monthly release newsletters, intelligent ticket routing, and PM task visibility — all triggered automatically based on defined conditions.

---

## Automations Built

| Automation | Tools Used | Description |
|---|---|---|
| [Jira Release Workflow](./jira-release-workflow.md) | Zapier, Jira | Pre-dev ticket setup for global vs pilot releases |
| [Monthly Release Newsletter](./newsletter-automation.md) | Zapier, Claude, Jira, Slack, HubSpot | Auto-drafts monthly release newsletter from product update data |
| [Ticket Auto-Routing](./ticket-routing.md) | Zapier, Jira, Slack | Routes new tickets to correct pod based on product area |
| [PM Slack Reminders](./slack-pm-reminders.md) | Zapier, Jira, Slack | Automated reminders for PM tasks and upcoming deadlines |

---

## Why I Built These

A significant chunk of PM time at scale gets consumed by operational overhead — manually triaging tickets, chasing stakeholders, formatting release notes, and keeping teams aligned. These automations reclaimed that time and improved operational efficiency across the team.

The goal was not to automate for its own sake — each automation was built to solve a specific, recurring friction point with a measurable impact on delivery speed and team visibility.
